# AES-256 Encryption

## Introduction

AES-256 (Advanced Encryption Standard with a 256-bit key) is a symmetric encryption algorithm widely used across the globe to secure sensitive data. It is recognized for its high level of security and efficiency, making it a preferred choice for encrypting data in various applications, including healthcare systems.

AES-256 is part of the AES family, which was established by the U.S. National Institute of Standards and Technology (NIST) in 2001. The "256" in AES-256 refers to the length of the encryption key used, which is 256 bits. This longer key length provides a higher level of security compared to AES-128 and AES-192.

## Implementation in DigiMedic EHR

In the DigiMedic EHR system, AES-256 is used to ensure the confidentiality and integrity of sensitive patient data. The implementation involves the following steps:

1. **Key Generation**: A secure 256-bit key is generated using a cryptographic random number generator.
2. **Encryption**: Data is encrypted using the AES-256 algorithm in Cipher Block Chaining (CBC) mode. This mode ensures that identical plaintext blocks will produce different ciphertext blocks, enhancing security.
3. **Decryption**: The encrypted data is decrypted using the same 256-bit key.

### Key Management

Key management is a critical aspect of the AES-256 implementation. In DigiMedic EHR, keys are securely stored and managed to prevent unauthorized access. The following practices are employed:

- **Key Storage**: Keys are stored in a secure key management system (KMS) that provides encryption and access control.
- **Key Rotation**: Keys are periodically rotated to minimize the risk of key compromise.
- **Access Control**: Access to encryption keys is restricted to authorized personnel only.

### Libraries and Tools

The DigiMedic EHR system uses the following libraries and tools for AES-256 encryption:

- **Python Cryptography Library**: A robust library that provides cryptographic recipes and primitives.
- **OpenSSL**: A widely-used library for secure communication and encryption.

## Usage Examples

### Encrypting Data

The following Python code snippet demonstrates how to encrypt data using AES-256 in the DigiMedic EHR system:

```python
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.primitives import padding
from cryptography.hazmat.backends import default_backend
import os

# Generate a random 256-bit key
key = os.urandom(32)

# Generate a random initialization vector (IV)
iv = os.urandom(16)

# Create a Cipher object
cipher = Cipher(algorithms.AES(key), modes.CBC(iv), backend=default_backend())

# Create an encryptor object
encryptor = cipher.encryptor()

# Pad the plaintext to be a multiple of the block size
padder = padding.PKCS7(algorithms.AES.block_size).padder()
plaintext = b"Sensitive patient data"
padded_data = padder.update(plaintext) + padder.finalize()

# Encrypt the padded plaintext
ciphertext = encryptor.update(padded_data) + encryptor.finalize()

print(f"Ciphertext: {ciphertext.hex()}")
```

### Decrypting Data

The following Python code snippet demonstrates how to decrypt data using AES-256 in the DigiMedic EHR system:

```python
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.primitives import padding
from cryptography.hazmat.backends import default_backend

# Use the same key and IV as for encryption
key = b'...'  # 256-bit key
iv = b'...'   # 128-bit IV

# Create a Cipher object
cipher = Cipher(algorithms.AES(key), modes.CBC(iv), backend=default_backend())

# Create a decryptor object
decryptor = cipher.decryptor()

# Decrypt the ciphertext
decrypted_padded_data = decryptor.update(ciphertext) + decryptor.finalize()

# Unpad the decrypted data
unpadder = padding.PKCS7(algorithms.AES.block_size).unpadder()
decrypted_data = unpadder.update(decrypted_padded_data) + unpadder.finalize()

print(f"Decrypted data: {decrypted_data.decode()}")
```
